## Project Overview

WP-Starter is a modern WordPress starter theme that removes jQuery dependency and leverages Laravel's Blade templating engine for improved developer experience. The theme has been modernized with Vite 7, Alpine.js v3.14.9 (bundled), Laravel 11 components, ACF Pro integration, TypeScript support, and comprehensive code quality tools.

## Key Technologies

- WordPress theme development
- Laravel Blade templating (via Illuminate v11 components)
- TailwindCSS v4 with @tailwindcss/postcss
- Vite 7 for ultra-fast asset compilation and HMR
- Alpine.js v3.14.9 for reactive components (replaced jQuery)
- PostCSS for CSS processing
- PHP 8.2+ with strict types and modern features
- ACF Pro with custom helpers and Blade directives
- TypeScript with full type definitions
- Code quality tools (ESLint, Prettier, PHPCS, PHPStan)

## Architecture Overview

### Directory Structure
- `config/` - Theme configuration and WordPress setup
  - `functions.php` - Bootstrap file that loads the application
  - `vite.php` - Vite integration helper
  - `app.php` - Application configuration
- `src/` - PHP source code with modern architecture
  - `Acf/` - ACF helper classes (Fields, Blocks, Options)
  - `Providers/` - Service providers (Blade, Menu, Theme, Security, ACF)
  - `types/` - TypeScript type definitions
  - `Application.php` - Main application class
  - `Config.php` - Configuration management
  - `Security.php` - CSP and security features
  - `helpers.php` - Helper functions
- `acf-json/` - ACF field group JSON sync
- `blocks/` - ACF Gutenberg blocks
- `resources/` - Source files for compilation
  - `css/` - TailwindCSS and custom styles
  - `js/` - TypeScript/JavaScript source files
- `templates/` - Blade templates (replaces traditional PHP templates)
  - `layouts/` - Base layouts
  - `partials/` - Reusable components
- `dist/` - Compiled assets (generated by Vite)
- `compiled/` - Blade template cache

### Blade Templating

This theme uses Laravel Blade instead of traditional WordPress templates. Key patterns:

- Templates extend layouts: `@extends('layouts.app')`
- Custom Blade components registered in `config/functions.php`
- The `@loop` component handles WordPress post loops
- All templates use `.blade.php` extension

### Asset Pipeline

Vite configuration (`vite.config.js`):
- Compiles `resources/css/app.css` → `dist/assets/app-[hash].css`
- Processes JavaScript from `resources/js/`
- Handles TailwindCSS v4 compilation
- Automatic versioning with content hashing
- Hot Module Replacement (HMR) in development
- Development server runs on `http://localhost:5173`

### Service Provider Architecture

The theme uses a modern service provider pattern:
- `BladeServiceProvider` - Manages Blade templating engine
- `MenuServiceProvider` - Registers navigation menus
- `ThemeServiceProvider` - Core theme functionality
- `SecurityServiceProvider` - CSP headers and security features
- `AcfServiceProvider` - ACF integration, blocks, and Blade directives

### Performance Optimizations

The theme implements several performance strategies:
- Vite's ultra-fast build system (100x faster incremental builds)
- Alpine.js bundled for lightweight reactivity (15KB vs jQuery's 87KB)
- Vite plugins: compression (gzip/brotli), image optimization, legacy support
- Bundle analysis for monitoring size
- Automatic code splitting and tree shaking
- Asset versioning through Vite manifest
- Deferred script loading
- Modern CSS with TailwindCSS v4's new performance engine

## Development Workflow

1. **CSS Changes**: Edit files in `resources/css/`, TailwindCSS v4 classes are available everywhere
2. **JavaScript**: Add/modify TypeScript/Alpine.js components in `resources/js/`
3. **Templates**: Create/edit Blade templates in `templates/`
4. **PHP Classes**: Add service providers or classes in `src/`
5. **Configuration**: Use `.env` file for local settings (copy from `.env.example`)
6. **Build Process**: Always run through Vite, never edit `dist/` directly

## Important Considerations

- The theme uses Alpine.js instead of jQuery - all interactive components should use Alpine
- Blade templates are compiled to `compiled/` - this directory can be cleared if needed
- The `.vite/manifest.json` file tracks versioned assets
- Configure local development in `.env` file (copy from `.env.example`)
- Theme text domain: `wp-starter`
- Namespace for PHP classes: `WordpressStarter\`
- PHP 8.2+ is required with strict types enabled
- All service providers extend the base `ServiceProvider` class
- ACF Pro is required for ACF features
- Use Local by Flywheel for Mac development

## Environment Configuration

The theme supports environment-based configuration:

## ACF Integration

The theme includes comprehensive ACF support:

### Blade Directives
- `@field('field_name')` - Output field value
- `@option('option_name')` - Output option field
- `@hasfield('field_name')...@endhasfield` - Conditional display
- `@repeater('field_name')...@endrepeater` - Loop through repeater
- `@flexible('field_name')...@endflexible` - Loop through flexible content
- `@layout('layout_name')...@endlayout` - Check flexible content layout
- `@group('field_name')...@endgroup` - Access group field

### Helper Functions
```php
use WordpressStarter\Acf\Fields;

// Get field with default
Fields::get('field_name', $postId, 'default');

// Get option with caching
Fields::option('option_name', 'default');

// Get image with size
Fields::image('image_field', 'large', $postId);

// Get link field
Fields::link('link_field', $postId);
```

### ACF Blocks
- Blocks are auto-discovered from `blocks/` directory
- Each block needs `block.json` and `template.blade.php`
- Blocks use Blade templates for rendering

### Available Blocks
The theme includes 13 pre-built ACF blocks:
- `accordion` - Expandable accordion with multiple items
- `cta` - Call to action section with title, content and button
- `divider` - Decorative divider with icon
- `four-columns` - Four column layout
- `hero` - Hero section with background image
- `image` - Single image with optional caption and border
- `one-column` - Single column content
- `one-third-two-thirds` - Asymmetric two column layout
- `three-columns` - Three column layout
- `two-columns` - Two equal columns
- `two-columns-images` - Two columns with images
- `two-thirds-one-third` - Asymmetric two column layout (reversed)
- `video` - Embed video from WordPress media or external source
